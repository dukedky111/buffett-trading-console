# 回测规则

## 1. 核心原则

回测必须避免未来函数。

只能使用信号发生时已经可获得的信息：

- 视频发布之后的数据。
- 当时已经公布的财报。
- 当时市场可见的价格。
- 当时 transcript 中明确出现的点位和判断。

## 2. 入场价格

默认规则：

```text
视频发布后第一个可交易日开盘价
```

如果视频在盘中发布：

```text
使用视频发布时间之后的下一根 K 线收盘价或下一日开盘价
```

## 3. 出场规则

出场只能来自事先定义的规则：

- 价格严重高于内在价值。
- 护城河破坏。
- 管理层诚信问题。
- 更好的机会。
- 达到组合风险控制条件。

不得因为回测结果事后优化卖点。

## 4. 必须记录的指标

| 指标 | 说明 |
| --- | --- |
| CAGR | 年化收益 |
| Max Drawdown | 最大回撤 |
| Win Rate | 胜率 |
| Average Win / Loss | 平均盈亏比 |
| Profit Factor | 总盈利 / 总亏损 |
| Holding Period | 平均持仓周期 |
| Excess Return | 相对 S&P 500 超额收益 |
| Signal Hit Rate | 视频信号命中率 |
| Filter Improvement | Buffett 过滤后是否改善表现 |

## 5. 关键对照组

至少比较三组：

```text
组 A：直接跟随视频信号
组 B：视频信号 + Buffett 质量过滤
组 C：视频信号 + Buffett 质量过滤 + 安全边际
```

如果 C 明显优于 A，说明系统有价值。

如果 A 优于 C，需要检查是否样本期偏向动量行情，或 Buffett 过滤过于保守。
